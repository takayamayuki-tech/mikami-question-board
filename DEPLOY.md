# デプロイ手順（コピペ＆クリックだけで完了します）

所要時間: **約5分**
やること: ① Google Apps Script を作る → ② コードを貼り付ける → ③ Web App として公開する → ④ URL を取得して三上さんが共有用に保存

---

## 事前準備

- スプレッドシート（`三上グルコン質問フォーム`）を **「リンクを知っている全員が閲覧可能」** に設定済みであること
- このリポジトリをローカルにクローン or ダウンロード済みであること

---

## ステップ 1: Google Apps Script プロジェクトを作る

1. ブラウザで <https://script.google.com/home> を開く（Googleアカウントにログインした状態で）
2. 左上の **「+ 新しいプロジェクト」** をクリック
3. プロジェクト名（左上の「無題のプロジェクト」のところ）を **`三上グルコン質問回答ボード`** に変更

---

## ステップ 2: コードを貼り付ける

### 2-1. `Code.gs` を貼り付ける

1. 左サイドの **`コード.gs`** をクリック
2. 既存の中身を **全部削除**
3. このリポジトリの `Code.gs` の中身を **全部コピーして貼り付け**

### 2-2. HTML ファイルを 3 つ追加する

GAS の左サイドにある **「ファイル」** の右の **「+」ボタン** → **「HTML」** を選んで、次の 3 ファイルを順番に追加します。
**ファイル名は拡張子なしで入力してください**（GAS が自動で `.html` を付けます）。

| 追加するファイル名 | 中身 |
|---|---|
| `Index` | このリポジトリの `Index.html` の中身を全部貼り付け |
| `Stylesheet` | このリポジトリの `Stylesheet.html` の中身を全部貼り付け |
| `JavaScript` | このリポジトリの `JavaScript.html` の中身を全部貼り付け |

### 2-3. （任意）`appsscript.json` のマニフェストを更新

これは省略しても動きますが、より確実にしたい場合：

1. GAS の左下の歯車アイコン **「プロジェクトの設定」** をクリック
2. **「『appsscript.json』マニフェストファイルをエディタで表示する」** にチェック
3. 左サイドに `appsscript.json` が表示されるのでクリック
4. このリポジトリの `appsscript.json` の中身を貼り付け

### 2-4. 保存

画面上部の **フロッピーディスクのアイコン**（または `Cmd + S` / `Ctrl + S`）で保存。

---

## ステップ 3: 動作確認

1. GAS エディタの上部にある関数選択ドロップダウンで **`debug_listFirstRows`** を選ぶ
2. **「実行」** ボタンをクリック
3. 初回は **権限の確認ダイアログ** が出ます:
   - **「権限を確認」** をクリック
   - Google アカウントを選ぶ
   - 「このアプリは Google で確認されていません」と出たら **「詳細」→「（プロジェクト名）に移動（安全ではないページ）」** をクリック
   - **「許可」** をクリック
4. 下部の **「実行ログ」** にスプレッドシートの先頭 5 行が表示されればOK

ログに何も出なかったり、エラーが出た場合は、`Code.gs` 上部の `SPREADSHEET_ID` がスプレッドシートのIDと一致しているか確認してください（既に設定済みのはずです）。

---

## ステップ 4: Web App として公開する

1. GAS エディタの右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 左の歯車アイコンをクリック → **「ウェブアプリ」** を選択
3. 設定:
   - **説明**: `三上グルコン質問回答ボード v1`（任意）
   - **次のユーザーとして実行**: **自分（自分のメールアドレス）**
   - **アクセスできるユーザー**: **全員**
4. **「デプロイ」** をクリック
5. 「ウェブアプリ」のURL（`https://script.google.com/macros/s/...../exec` の形式）が表示されるので **コピーして保管**

このURLが、PC・スマホ・誰にでも送れる **共有URL** です。

---

## ステップ 5: 動作確認

コピーしたURLをブラウザで開き、質問カードが表示されることを確認してください。

- 「回答済みにする」ボタンを押すと、その質問が下のセクション（回答済み）に灰色になって移動する
- スプレッドシートを開くと、新しく `回答済み` `回答日時` の列が追加されていて、押した質問の行が `TRUE` になっている
- ブラウザのページを再読み込みしても、回答済みの状態が保持される（スプレッドシートに保存されているため）

---

## 更新の反映について

- スプレッドシートに新しい質問が追加されたら、ボードの **「更新」** ボタン（または30秒ごとの自動更新）で取り込まれます
- コード（`Code.gs` や HTML）を変更したい時:
  1. GAS で編集 → 保存
  2. **「デプロイ」→「デプロイを管理」**
  3. 既存のデプロイの右の **鉛筆アイコン**（編集）をクリック
  4. **バージョンを「新しいバージョン」** に変更 → **「デプロイ」**
  5. URL は変わらないのでそのまま使えます

---

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「あなたには編集権限がありません」と出る | スプレッドシートの所有者と、GASを動かしているGoogleアカウントが同じか確認 |
| 質問が表示されない | `Code.gs` 上部の `FILTER_FROM` を `new Date('2024-01-01T00:00:00+09:00')` などに緩めて試す |
| スプレッドシートのヘッダー名が想定と違う | `Code.gs` 上部の `HEADER_PATTERNS` に正規表現を追加して再デプロイ |
| 列の検出がおかしい | GAS で `debug_getQuestions` を実行 → ログを見て、どの列がどう取れているか確認 |

## 共有

ステップ4で取得した URL（`https://script.google.com/macros/s/.../exec`）を、三上さんがブックマーク・LINE保存・メモ等に保存してください。
PC・スマホ問わず、その URL をブラウザで開けばすぐ見られます。
